Angular Components Cheatsheet

What is components in Angular ?

Components are the building blocks of Angular applications. They are reusable pieces of UI that encapsulate a specific functionality and can be nested within each other.

Component Creation

To create a new component using the Angular CLI, run the following command:

ng generate component my-component

Output: This will generate a new component with four files: `.ts`, `.html`, `.css`, and `.spec.ts`.

Key components of a component:
  • Template: The HTML structure of the component.
  • Styles: The CSS styles that apply to the component.
  • Logic: The TypeScript code that defines the component's behavior.

Benefits of using components:
  • Reusability: Components can be reused in multiple places within your application.
  • Modularity: They help organize your code into smaller, more manageable units.
  • Maintainability: Components are easier to update and maintain.

By understanding and effectively using components, you can create well-structured and maintainable Angular applications.

@Input Decorator

The @Input() decorator allows you to pass data from a parent component to a child component.

<app-child [childData]="parentData"></app-child>

Output: The childData will be passed from the parent component to the child component and can be used within the child component.

@Output Decorator

The @Output() decorator allows the child component to emit events back to the parent component.

<app-child (notify)="onNotify($event)"></app-child>

Output: When the child component emits the notify event, the parent component reacts by executing the onNotify() function.
